# Graphs and Trees

## 1. Graphs (Network Relationships)

* **Definition:** A non-linear data structure used to model complex network relationships between entities. It consists of a finite set of **Nodes (or Vertices)** representing objects, connected together by lines called **Edges**.
* **Analogy:** Think of a flight map for an airline network. Each airport city is a **Node**, and the non-stop flight routes connecting cities together are the **Edges**.

### Graph Classifications

* **Directed vs. Undirected:** In a *Directed* graph (like Twitter), connections are one-way streets (Alice follows Bob, but Bob might not follow Alice). In an *Undirected* graph (like Facebook), connections are two-way streets (a friendship is mutual).
* **Weighted vs. Unweighted:** In a *Weighted* graph, edges hold values representing costs like physical distance, travel time, or toll fees (e.g., Google Maps routing).

## 2. Trees (Hierarchical Data Structures)

* **Definition:** A specialized type of connected graph containing no closed loops or cycles, organized in a strict top-down parent-child hierarchy.
* **Analogy:** Think of a corporate organizational chart. The CEO sits at the absolute peak (**Root Node**). Vice Presidents branch out directly below them (**Child Nodes**), and frontline individual contributors with no subordinates sit at the very bottom tips (**Leaf Nodes**).

### Binary Search Trees (BST)

A highly optimized variation where every node has a maximum of two children, enforcing a strict ordering rule:

* Values **smaller** than the parent go into the **Left Subtree**.
* Values **larger** than the parent go into the **Right Subtree**.

```python
class BSTNode:
    """A single node structure inside a Binary Search Tree."""
    def __init__(self, key):
        self.val = key      # The data value payload
        self.left = None    # Left pointer channel for smaller numbers
        self.right = None   # Right pointer channel for larger numbers

def insert_bst(root, key):
    """Enforces the BST ordering rule dynamically during insertion."""
    if root is None:
        return BSTNode(key)
    
    if key < root.val:
        # Route left recursively if value is smaller
        root.left = insert_bst(root.left, key)
    else:
        # Route right recursively if value is equal or larger
        root.right = insert_bst(root.right, key)
    return root

```

## 3. Tree Traversals (In-Order, Pre-Order, Post-Order)

* **Definition:** The programmatic strategy used to systematically visit every single node in a tree structure exactly once.
* **Analogy:** Imagine a supervisor touring a three-wing office building to collect reports.
* **Pre-Order** is like signing documents at the main entrance desk *before* walking down the sub-hallways.
* **In-Order** is like walking sequentially from left-wing to center-hub to right-wing, which gracefully reads items in sorted numerical order.
* **Post-Order** is like gathering paperwork from all sub-clerks first, and only signing off on the master project folder once all deep children have finished.



```python
# Given a root node, these recursive functions read tree data via different paths

def print_in_order(root):
    """Left -> Root -> Right. (Extracts BST keys in perfect ascending order!)"""
    if root:
        print_in_order(root.left)
        print(root.val, end=" ")
        print_in_order(root.right)

def print_pre_order(root):
    """Root -> Left -> Right. Perfect for duplicating or copying trees."""
    if root:
        print(root.val, end=" ")
        print_pre_order(root.left)
        print_pre_order(root.right)

def print_post_order(root):
    """Left -> Right -> Root. Perfect for deleting nodes or computing folder sizes."""
    if root:
        print_post_order(root.left)
        print_post_order(root.right)
        print(root.val, end=" ")

```

## 4. Heaps & Priority Queues

* **Definition:** A **Heap** is a specialized tree-based complete data structure that maintains an absolute extreme value at its root node. A **Priority Queue** is an abstract container where elements are served based on an explicit priority ranking score rather than their arrival sequence time.
* **Analogy:** Think of an emergency room (ER) triage waiting lounge. Patients do not get treated based on who walked through the front door first (traditional FIFO queue). Instead, a patient arriving with a critical life-threatening condition is instantly pushed right to the front of the care stream (**Root priority position**) over patients waiting with minor scrapes.

[Image layout demonstrating a Min-Heap binary tree layout mapping the absolute minimum value safely at the top root node]

```python
# Python implements heaps using standard lists optimized via the built-in 'heapq' module
import heapq

# Creating an empty Min-Heap container list structure
priority_triage = []

# Pushing tuples format: (priority_score, data_payload)
# Lower numbers mean higher priority rank in a Min-Heap
heapq.heappush(priority_triage, (3, "Patient with Broken Arm"))
heapq.heappush(priority_triage, (1, "Patient with Heart Distress")) # Root assignment!
heapq.heappush(priority_triage, (2, "Patient with Deep Laceration"))

# PEEK: Review the most critical item in constant time O(1)
print(f"Next critical case: {priority_triage[0]}") 

# EXTRACT MIN: Pop the top item out in logarithmic time O(log n)
served_case = heapq.heappop(priority_triage)
print(f"Actively treating: {served_case[1]}") # Output: Patient with Heart Distress

```
