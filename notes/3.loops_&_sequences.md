# Loops and Sequences

## 1. Lists (Ordered & Mutable Sequences)

* **Definition:** A list is an ordered, changeable (mutable) sequence container that can hold a mix of different data types (strings, numbers, or even other lists) using a zero-based position index.
* **Analogy:** Think of a list as a modular train where each cargo car holds an item. You can look up what's inside a car by counting from the front (`0, 1, 2...`), swap out the cargo inside an existing car, unhook cars to delete items, or couple new cargo cars to the very back of the train (`.append()`).

[Image diagram showing a list sequence with positive zero-based indexes pointing from left to right and negative indexes pointing from right to left]

```python
# Creating a list and accessing elements via index
cities = ['Los Angeles', 'London', 'Tokyo']
print(cities[0])   # Output: 'Los Angeles' (First item)
print(cities[-1])  # Output: 'Tokyo' (Negative indexing starts from the back)

# Modifying (Mutating) an element in place
cities[1] = 'Paris'

# Core List Operations
cities.append('Nairobi')      # Adds to the very end
cities.insert(1, 'New York')  # Inserts at index 1, shifting others right
cities.remove('Tokyo')        # Removes item by value name
popped_city = cities.pop(0)   # Removes and returns item at index 0

print(cities) # Output: ['New York', 'Paris', 'Nairobi']

```

## 2. Tuples (Ordered & Immutable Sequences)

* **Definition:** A tuple is an ordered, unchangeable (immutable) sequence container. Once a tuple is created, you cannot add, remove, or modify its values.
* **Analogy:** Think of a tuple as a collection of items sealed inside a glass display case. You can look at the items and read their positions perfectly, but you are physically blocked from changing, rearranging, or replacing any item inside the case without buying an entirely new case.

```python
# Creating a tuple (Uses parentheses instead of square brackets)
coordinates = (34.0522, -118.2437)

# Reading data works exactly like a list
print(coordinates[0])  # Output: 34.0522

# Python safety check: The line below will raise a TypeError crash!
# coordinates[0] = 40.7128  # Immutability blocks changes!

# Tuple Unpacking: Extracting values into standalone variables in one line
latitude, longitude = coordinates
print(f"Lat: {latitude}, Lon: {longitude}")

```

## 3. The `range()` Function

* **Definition:** A built-in generator sequence that efficiently produces a progression of integers based on a defined `(start, stop, step)` configuration layout.
* **Analogy:** Imagine a factory machine programmed to punch ticket numbers. Instead of printing millions of tickets ahead of time and wasting warehouse space, the machine only stamps out the next consecutive number precisely when a customer stands at the gate and requests it.

```python
# Form A: range(stop) -> Generates numbers from 0 up to (but excluding) stop
print(list(range(5)))  # Output: [0, 1, 2, 3, 4]

# Form B: range(start, stop)
print(list(range(2, 6)))  # Output: [2, 3, 4, 5]

# Form C: range(start, stop, step)
# Generates numbers from 1 to 9, skipping by 2 each time
odd_numbers = list(range(1, 10, 2))
print(odd_numbers)  # Output: [1, 3, 5, 7, 9]

```

## 4. `for` Loops vs. `while` Loops

* **Definition:** **`for` loops** step through a predetermined sequence of items one by one until the collection is exhausted. **`while` loops** repeat a block of code continuously as long as a target boolean condition remains `True`.
* **Analogy:** * A **`for` loop** is like a dealer passing out cards to a fixed line of 5 players at a table. You know exactly when it starts and when it ends (after the 5th player).
* A **`while` loop** is like an endurance driver doing laps on a race track. They keep driving around and around *while* they still have fuel in the tank. The moment the tank hits empty, they stop.



[Image diagram showing the execution flow layout of a For loop repeating over a collection versus a While loop checking a condition switch]

```python
# FOR LOOP: Iterating safely over a list collection
shopping_list = ['milk', 'bread', 'eggs']
for item in shopping_list:
    print(f"Purchasing: {item}")

# WHILE LOOP: Running based on a changing condition counter
countdown = 3
while countdown > 0:
    print(countdown)
    countdown -= 1  # Crucial: Must update condition to avoid an infinite loop!
print("Blast off!")

```

## 5. Loop Controls (`break` and `continue`)

* **Definition:** **`break`** terminates the loop immediately, jumping completely out of its block. **`continue`** skips the rest of the current iteration loop cycle and jumps straight to the top of the next loop pass.
* **Analogy:** Imagine auditing an assembly line of boxes.
* **`continue`**: You spot a recycling box, skip the sorting step for it, and instantly reach for the next box in line.
* **`break`**: You suddenly detect a hazardous fire alarm on the line. You drop everything and run out of the factory building immediately, completely canceling the rest of the workday.



```python
# Testing Break & Continue controls
numbers = [1, 2, 3, 4, 5]

for num in numbers:
    if num == 3:
        continue  # Skips printing '3' and jumps straight to number 4!
    if num == 5:
        break     # Shuts down the entire loop immediately!
    print(f"Processing number: {num}")

# Output:
# Processing number: 1
# Processing number: 2
# Processing number: 4

```

## 6. List Comprehensions

* **Definition:** A compressed, elegant syntax format used to create a brand-new list from an existing iterable by running an expression loop on one line.
* **Analogy:** Imagine a factory sorting line. Instead of taking a box, writing out instructions on a clipboard, opening the box, changing the item, and putting it in a new box (**traditional loop**), you install an inline transformation tunnel over the conveyor belt that modifies every passing item instantly.

```python
# Traditional multi-line approach:
squares_traditional = []
for x in range(1, 4):
    squares_traditional.append(x ** 2)

# Advanced production standard: List Comprehension
# Syntax: [expression for item in iterable]
squares_modern = [x ** 2 for x in range(1, 4)]
print(squares_modern)  # Output: [1, 4, 9]

# Comprehension with an inline IF filter condition
even_squares = [x ** 2 for x in range(1, 6) if x % 2 == 0]
print(even_squares)  # Output: [4, 16] (Only evaluates 2 and 4)

```

## 7. Lambda (Anonymous) Functions

* **Definition:** Small, single-line, unnamed (anonymous) functions defined using the `lambda` keyword that can take any number of inputs but compute only one single expression.
* **Analogy:** Think of a regular function like an industrial multi-use kitchen food processor that you bolt to the counter. A lambda function is like a disposable plastic fruit juicer tool. You use it inline once for a single specific task, and discard it immediately instead of storing it.

```python
# Syntax: lambda parameters: expression

# Best Practice: Use them inline inside functional processors like map() or filter()
numbers = [1, 2, 3, 4, 5]

# 1. MAP: Applies the lambda operation to EVERY item in the sequence
squared_nums = list(map(lambda num: num ** 2, numbers))
print(squared_nums)  # Output: [1, 4, 9, 16, 25]

# 2. FILTER: Keeps only the items where the lambda condition evaluates to True
even_nums = list(filter(lambda num: num % 2 == 0, numbers))
print(even_nums)     # Output: [2, 4]

```
