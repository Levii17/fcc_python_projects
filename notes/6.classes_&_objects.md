# Classes and Objects

## 1. Classes vs. Objects

* **Definition:** A **Class** is a user-defined blueprint or template that outlines the structure of data and behavior. An **Object** is a specific instance created from that blueprint, holding actual real-world data.
* **Analogy:** Think of a **Class** as an architectural blueprint for a house. It isn't a house itself, but it details where the walls and doors go. An **Object** is the actual physical house built using that blueprint. You can build 50 individual houses (Objects) from a single blueprint (Class), and each can have different families living inside them.

```python
# The Blueprint (Class)
class Dog:
    # The initializer that sets up the dog's data
    def __init__(self, name, age):
        self.name = name  # Instance data
        self.age = age    # Instance data

    # The behavior the dog can perform
    def bark(self):
        print(f"{self.name.upper()} says woof woof! I'm {self.age} years old!")

# The Real-World Instances (Objects)
dog_1 = Dog("Jack", 3)       # Creating dog instance 1
dog_2 = Dog("Thatcher", 5)   # Creating dog instance 2

# Making the objects take action
dog_1.bark()  # Output: JACK says woof woof! I'm 3 years old!
dog_2.bark()  # Output: THATCHER says woof woof! I'm 5 years old!

```

## 2. Attributes (Instance vs. Class)

* **Definition:** **Attributes** are variables tied to classes or objects that store information. **Instance Attributes** are unique to each individual object, while **Class Attributes** are shared universally across every single object created from that class.
* **Analogy:** If our class is `Car`, an *Instance Attribute* is the **color** or **license plate** (unique to each car). A *Class Attribute* is something like **number_of_wheels = 4** (applies universally to all cars off the assembly line).

```python
class Car:
    # Class Attribute: Shared by ALL cars
    wheels = 4

    def __init__(self, color, model):
        # Instance Attributes: Unique to each individual car
        self.color = color
        self.model = model

# Accessing class attributes directly from the Class
print(Car.wheels)  # Output: 4

# Creating individual car objects
car_1 = Car("red", "Toyota Corolla")
car_2 = Car("green", "Lamborghini Revuelto")

# Accessing unique instance data via dot notation
print(car_1.model)  # Output: Toyota Corolla
print(car_2.color)  # Output: red
print(car_2.wheels) # Output: 4 (Inherited from the class)

```

## 3. Methods

* **Definition:** **Methods** are functions defined inside a class that allow an object to perform actions, interact with, or modify its own internal data.
* **Analogy:** If an object is a smart microwave, the attributes are its size and brand. The **Methods** are the buttons on the front panel—like `start()`, `stop()`, or `defrost()`. They are the explicit actions the microwave knows how to perform.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

    # A method that uses internal data to perform an action
    def describe(self):
        return f"This car is a {self.color} {self.model}."

# Creating the object
my_ride = Car("black", "Tesla Model 3")

# Invoking the action
print(my_ride.describe())  # Output: This car is a black Tesla Model 3.

```

## 4. Special (Dunder) Methods

* **Definition:** **Special Methods** (or **Magic/Dunder Methods**) are built-in hooks that start and end with double underscores (`__`). They allow your custom objects to hook into Python's native operators and behaviors, like addition (`+`), length checks (`len()`), or print statements.
* **Analogy:** Imagine buying a custom steering wheel for a video game console. For the console to recognize when you step on the gas or turn the wheel, your device must map to the game console's pre-existing, universal controller protocols. Dunder methods are Python's universal protocol maps.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    # Defines what happens when you call str(book_object) or print()
    def __str__(self):
        return f"'{self.title}' ({self.pages} pages)"

    # Defines what happens when you call len(book_object)
    def __len__(self):
        return self.pages

    # Defines what happens when you use the '==' comparison operator
    def __eq__(self, other):
        return self.pages == other.pages

# Testing our custom native features
book1 = Book("Build Wealth Like a Boss", 420)
book2 = Book("Be Your Own Star", 420)

print(str(book1))    # Output: 'Build Wealth Like a Boss' (420 pages)
print(len(book1))    # Output: 420
print(book1 == book2) # Output: True (Because they have the same page count)

```

## 5. Dynamic Attribute Handling

* **Definition:** Dynamic attribute handling is the ability to read, write, check, or delete an object's properties at runtime using data strings, rather than hardcoding names directly into your script ahead of time.
* **Analogy:** Imagine a smart storage locker. Normally, you open a locker with a specific fixed physical key (like typing `user.name`). Dynamic handling is like a digital touchscreen keypad where you type in *any* text string command—like "Locker_12"—and the locker system dynamically matches your text entry to find the corresponding physical box.

### The Four Key Functions:

* `hasattr(obj, 'name')`: Checks if an attribute exists (**True**/**False**).
* `getattr(obj, 'name')`: Reads the value of an attribute dynamically.
* `setattr(obj, 'name', value)`: Creates or updates an attribute dynamically.
* `delattr(obj, 'name')`: Deletes an attribute from the object dynamically.

```python
class Configuration:
    pass  # Starting with a completely empty blueprint

# Simulating raw external data fetched from a JSON file or User Input
external_settings = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30
}

config = Configuration()

# 1. SETATTR: Dynamically assigning properties from dictionary strings
for key, val in external_settings.items():
    setattr(config, key, val)

# 2. HASATTR & GETATTR: Dynamically checking and reading properties safely
requested_feature = 'server_url'

if hasattr(config, requested_feature):
    # Read the data dynamically using the text variable
    data = getattr(config, requested_feature)
    print(f"Found {requested_feature}: {data}") # Output: Found server_url: https://api.example.com
else:
    print("Property not found!")

# 3. DELATTR: Dynamically removing sensitive properties to clean up memory
delattr(config, 'timeout_sec')
print(hasattr(config, 'timeout_sec'))  # Output: False

```
