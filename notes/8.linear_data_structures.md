# Linear Data Structures

## 1. Algorithms & Big O Notation

* **Definition:** An **algorithm** is a precise, unambiguous set of step-by-step instructions designed to execute in a finite number of steps to solve a problem. **Big O Notation** describes the worst-case performance or resource growth rate (time or memory space) of an algorithm as the size of the input ($n$) scales up.
* **Analogy:** Think of an algorithm as a cooking recipe. If you have a recipe to bake 1 loaf of bread versus 1 million loaves of bread, the basic instructions are the same, but the time and kitchen space required will scale drastically. Big O notation tells you exactly how much your kitchen operations will explode as that order size grows.

### Common Time Complexities (From Fastest to Slowest)

1. **$O(1)$ — Constant Time:** Takes the exact same running time regardless of the size of the input data.
2. **$O(\log n)$ — Logarithmic Time:** Execution time grows very slowly because the algorithm continually cuts the problem size in half with each operation (e.g., Binary Search).
3. **$O(n)$ — Linear Time:** Execution time grows in direct, equal proportion to the size of the input data.
4. **$O(n \log n)$ — Log-Linear Time:** The baseline standard for highly efficient sorting algorithms like Merge Sort and Quick Sort.
5. **$O(n^2)$ — Quadratic Time:** Execution time scales quadratically; common in nested loop setups. Highly inefficient for massive data scales.

# --- O(1) Constant Time Example ---
# No matter how large the number is, it takes only 1 arithmetic evaluation step.
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'

# --- O(n) Linear Time Example ---
# If the grades list size doubles, the loop operation count doubles.
def print_all_grades(grades):
    for grade in grades:  
        print(grade)

# --- O(n²) Quadratic Time Example ---
# The inner loop runs 'n' times for every single iteration of the outer loop ($n \times n = n^2$).
def quadratic_nested_loops(n):
    for i in range(n):
        for j in range(n):
            print("Hello, World!")

## 2. Dynamic Arrays vs. Static Arrays

* **Definition:** **Static Arrays** have a fixed capacity determined at instantiation that cannot be altered during runtime. **Dynamic Arrays** are flexible structures that grow or shrink automatically behind the scenes by allocating a larger memory block and copying elements over when they hit capacity limit thresholds.
* **Analogy:** * A **Static Array** is like an airplane cabin rows block. Once the structural row seats are built, you cannot add seats mid-flight. To seat more people, you must purchase a completely separate, larger aircraft.
* A **Dynamic Array** is like an accordion file folder expansion suite. As you slide more papers in, the folder expands automatically to adapt.



```python
# NOTE: Python doesn't feature traditional primitive static arrays natively. 
# Python Lists function directly as highly optimized Dynamic Arrays.

# Initializing a Dynamic Array (Python List)
numbers = [3, 4, 5, 6]

# Core Operations & Complexities:
print(numbers[0])        # Access via Index: O(1) Constant Time
numbers.append(7)        # Append to End: Amortized O(1) Constant Time
numbers.insert(1, 99)    # Insert in Middle: O(n) Linear Time (shifts all subsequent elements right)
numbers.pop(2)           # Remove at Index: O(n) Linear Time (shifts remaining elements left)

```

## 3. Stacks vs. Queues

* **Definition:** A **Stack** is a **Last-In, First-Out (LIFO)** data container where items are added and removed strictly from one single boundary called the *Top*. A **Queue** is a **First-In, First-Out (FIFO)** sequential container where items enter at the *Back* (Enqueue) and leave from the *Front* (Dequeue).
* **Analogy:** * A **Stack** is like a heavy pile of dining plates on a kitchen counter. You can only stack a clean plate directly onto the top, and you must take off the topmost plate first to avoid smashing the layout.
* A **Queue** is like a physical line of grocery buyers waiting at a store checkout counter. The customer who arrived first gets processed and leaves first, while new shoppers join at the back of the line.



[Image layout illustrating a LIFO Stack showing push/pop operations at the top versus a FIFO Queue showing elements enqueuing at the back and dequeuing from the front]

```python
# --- STACK IMPLEMENTATION (Using standard list) ---
stack = []
stack.append("Plate 1")  # "Push" operation: O(1)
stack.append("Plate 2")  
popped_item = stack.pop() # "Pop" operation: O(1) (Removes from top/end)


# --- QUEUE IMPLEMENTATION (Using collections.deque for efficient O(1) operations) ---
from collections import deque
queue = deque()

queue.append("Customer 1") # "Enqueue" operation: O(1) (Adds to back)
queue.append("Customer 2")
served_customer = queue.popleft() # "Dequeue" operation: O(1) (Removes from front)

```

## 4. Singly Linked Lists

* **Definition:** A **Linked List** is a linear structural chain of independent **Nodes**. Each Node acts as a self-contained object housing its own data payload along with a memory pointer pointer reference that points directly to the next Node object sequence in line.
* **Analogy:** Imagine a neighborhood scavenger hunt game. You aren't given a map showing all house locations at once. Instead, you go to the starting address (**Head Node**). There you open a clue box containing your reward token (**Data**) and a slip of paper writing out the exact street address for the next hidden box sequence location (**Next Pointer reference**).

[Image representing a Singly Linked List sequence chain showing connected Nodes mapping data blocks with next-pointer arrows pointing forward to the Tail]

```python
class Node:
    """A fundamental structural unit node of a Singly Linked List."""
    def __init__(self, data):
        self.data = data      # Stores the target element payload
        self.next = None      # Link pointer tracking the next node object address reference

class SinglyLinkedList:
    def __init__(self):
        self.head = None      # The initial entry doorway node into the sequence chain

    def insert_at_start(self, new_data):
        """Prepends a node to the start. Complexity: O(1) Constant Time."""
        new_node = Node(new_data)
        new_node.next = self.head  # Connect new node to point to current head
        self.head = new_node       # Reassign head status onto our new node

    def insert_at_end(self, new_data):
        """Appends a node to the very end. Complexity: O(n) Linear Time."""
        new_node = Node(new_data)
        if not self.head:
            self.head = new_node
            return
        
        # Traverse forward node-by-node until hitting the Tail boundary
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node    # Link old tail node pointer to attach the new node

```
