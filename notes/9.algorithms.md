## 1. Linear Search vs. Binary Search

* **Definition:** **Linear Search** sequentially scans every single item in a collection from start to finish until it finds a target value. **Binary Search** is an optimized search algorithm that continuously divides a *strictly sorted* list in half, instantly discarding the half that cannot contain the target.
* **Analogy:** Imagine trying to find a specific word in a physical dictionary.
* **Linear Search** is like turning to page 1 and reading every single word word-by-word until you find it. If the word starts with "Z", it will take you all day.
* **Binary Search** is like opening the dictionary exactly to the middle. If your word alphabetically comes before the middle page, you flip to the middle of the left half, completely ignoring the entire right half of the book.



[Image diagram contrasting Linear Search scanning every single index versus Binary Search splitting a sorted list in half repeatedly]

```python
# --- Linear Search: Complexity O(n) Linear Time ---
# Works on unsorted or sorted lists alike.
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i  # Return index location if found
    return -1         # Return -1 if target does not exist


# --- Binary Search: Complexity O(log n) Logarithmic Time ---
# MANDATORY REQUIREMENT: The input array MUST be pre-sorted!
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  # Find the exact middle index
        
        if arr[mid] == target:
            return mid           # Target found!
        elif arr[mid] < target:
            low = mid + 1        # Discard left half
        else:
            high = mid - 1       # Discard right half
    return -1

```

## 2. Bubble Sort (The Iterative Swapping Sort)

* **Definition:** A simple sorting algorithm that repeatedly steps through a list, compares adjacent elements, and swaps them if they are in the wrong order. The algorithm repeats this process until the entire list is fully sorted.
* **Analogy:** Imagine a row of bubbles rising in a glass of carbonated soda. The largest, heaviest bubbles slowly push past the smaller ones, floating upward one position at a time until the biggest bubble reaches the very top of the liquid.

```python
# Complexity: O(n²) Quadratic Time (Highly inefficient for large datasets)
def bubble_sort(arr):
    n = len(arr)
    # Loop through all elements in the array
    for i in range(n):
        # Last i elements are already in place, no need to re-check them
        for j in range(0, n - i - 1):
            # Compare adjacent items
            if arr[j] > arr[j + 1]:
                # Swap them in place using Python's clean tuple assignment
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

```

## 3. Merge Sort (The Divide and Conquer Sort)

* **Definition:** A highly efficient, stable sorting algorithm that breaks a list down into sub-lists containing only a single element each, and then repeatedly merges those sub-lists back together in sorted order.
* **Analogy:** Imagine an office manager handed a massive stack of mixed, un-sorted tax files. Instead of trying to sort them all alone, the manager divides the stack in half and hands them to two supervisors. The supervisors divide them again down to individual clerks. When each clerk has just one folder (sorted by default), they cleanly merge their piles back up the chain, pairing elements sequentially until a single, perfectly sorted stack returns to the manager.

[Image illustrating the Divide and Conquer visual tree layout of Merge Sort showing a split down to single items and a sorted recombining phase]

```python
# Complexity: O(n log n) Log-Linear Time (Highly efficient, production standard)
def merge_sort(arr):
    # Base case: A list with 1 or 0 items is already sorted
    if len(arr) <= 1:
        return arr

    # DIVIDE: Split the array right down the middle
    mid = len(arr) // 2
    left_half = merge_sort(arr[:mid])   # Recursive split left
    right_half = merge_sort(arr[mid:])  # Recursive split right

    # CONQUER & MERGE: Combine sorted sub-lists back together
    sorted_list = []
    i = j = 0

    # Compare elements head-to-head from both halves
    while i < len(left_half) and j < len(right_half):
        if left_half[i] <= right_half[j]:
            sorted_list.append(left_half[i])
            i += 1
        else:
            sorted_list.append(right_half[j])
            j += 1

    # Clean up and append any remaining items left over from either half
    sorted_list.extend(left_half[i:])
    sorted_list.extend(right_half[j:])
    
    return sorted_list

```
